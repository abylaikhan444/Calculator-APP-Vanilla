# 🧮 Calculator App (Калькулятор на чистом JS)

Современный веб-калькулятор с интуитивно понятным интерфейсом, разработанный на чистом JavaScript (Vanilla JS). Проект демонстрирует навыки проектирования логики UI, работы с состояниями приложения (State), обработки пользовательских событий и решения классических вычислительных багов.

---

## 🛠️ Функционал приложения

* **Базовые арифметические операции:** Сложение (`+`), вычитание (`-`), умножение (`×`) и деление (`÷`).
* **Двухэкранный UI интерфейс:** Верхний дисплей (`write-box`) отображает историю текущего вычисления, а нижний (`result-box`) показывает вводимое число или итоговый результат.
* **Умный ввод точки:** Реализована валидация ввода — пользователь не может ввести две точки в одном числе (защита от формата `5.5.5`).
* **Полный сброс (AC):** Кнопка мгновенной очистки всех регистров памяти приложения и сброса состояния к начальному виду.
* **Цепочки вычислений:** Возможность делать непрерывные вычисления без нажатия на кнопку `=` (например, при вводе `5 + 5 + ...` калькулятор автоматически посчитает промежуточный результат).
* **Безопасность вычислений:** Добавлена валидация деления на ноль с выводом понятного предупреждения «Ошибка» вместо системного `Infinity`.

---

## 💻 Технологический стек

* **HTML5** — семантическая разметка сетки калькулятора.
* **CSS3** — стилизация кнопок, адаптивная сетка (Grid/Flexbox) и эффекты нажатия.
* **JavaScript (ES6+)** — управление состоянием, обработка кликов, ветвления `switch-case` и математическая логика.

## 🧠 Сложные задачи, которые я решил в проекте

1.  **Победил баг округления (IEEE 754):** В процессе тестирования столкнулся с классической проблемой JavaScript, когда `0.1 + 0.2` выдает `0.30000000000000004`. Решил этот баг с помощью комбинации методов `toFixed(10)` для отсечения микро-хвостов в двоичной системе и `parseFloat()` для удаления лишних нулей на конце целых чисел.
2.  **Принцип DRY (Don't Repeat Yourself) на практике:**
    Вместо создания раздутой конструкции `if/else` для каждой из 10 цифровых кнопок, я написал одну универсальную функцию `appendNumber(number)`, которая динамически принимает текстовое содержимое нажатой кнопки и склеивает строки.
3.  **Архитектура конечного автомата (State Management):**
    Успешно спроектировал логику хранения состояний через 3 независимые переменные: `currentInput` (текущий ввод), `previousInput` (число в памяти) и `operation` (активный знак). Это позволило разделить вычисления и отрисовку интерфейса.
4.  **Управление потоком в Switch-Case:**
    Разобрался с поведением оператора `switch` и важностью использования ключевого слова `break` для предотвращения нежелательного «проваливания» (fall-through) логики в соседние математические операции.
